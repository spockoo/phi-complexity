# Contribution à Phi-Complexity

Merci de l'intérêt que vous portez à **Phi-Complexity**. En tant que projet à la croisée des mathématiques et du développement logiciel, nous accueillons avec enthousiasme les contributions qui améliorent la précision, l'efficacité ou la documentation de nos outils.

---

## 🔬 Principes de Contribution

### 1. Rigueur Mathématique
Toute modification des algorithmes de calcul de complexité doit être accompagnée, si possible, d'une explication théorique ou d'une référence à une publication scientifique dans la description de la Pull Request.

### 2. Le Bot de Synchronisation
Le projet utilise un système d'évolution automatisé via `engine.py`. 
* Ne modifiez pas directement les fichiers générés automatiquement par le bot.
* Si vous souhaitez modifier la logique d'évolution, proposez vos changements sur le fichier `engine.py` lui-même.

---

## 🛠️ Comment contribuer ?

### Signaler un bug
Si vous trouvez une erreur de calcul ou un bug logiciel, ouvrez une **Issue** en précisant :
* Votre environnement (version de Python).
* Les données d'entrée ayant causé l'erreur.
* Le résultat attendu vs le résultat obtenu.

### Proposer des changements (Pull Requests)
1. **Forkez** le dépôt : [spockoo/phi-complexity](https://github.com).
2. **Créez une branche** pour votre fonctionnalité (`git checkout -b feature/ma-nouvelle-mesure`).
3. **Commitez** vos changements avec des messages clairs.
4. **Poussez** votre branche et ouvrez une **Pull Request**.

### Normes de Code
* Respectez la norme **PEP 8** pour le code Python.
* Documentez vos fonctions avec des "docstrings" claires (particulièrement pour les formules mathématiques).
* Assurez-vous que vos changements ne cassent pas le workflow GitHub Actions existant.
